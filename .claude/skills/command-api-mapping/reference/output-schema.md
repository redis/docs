# Output schema & pipeline

## File locations

| Artifact | Path | Role |
|---|---|---|
| Per-command files | `data/command-api-mapping/<COMMAND>.json` | **Source of truth** — author/edit these |
| Merged file | `data/command-api-mapping.json` | Generated by the merge script; consumed by layouts |
| Overrides | `data/command-api-mapping-overrides.json` | Replaces (not merges) generated entries |
| Merge script | `build/merge-command-api-mapping.sh` | Concatenates per-command files → merged file |
| Process doc | `build/command-api-mapping-README.md` | Short README for the live pipeline |
| Consumers | `layouts/partials/components/api-methods-tab.html`, `layouts/partials/commands-foldout.html` | Render the tables |

Filename = command name verbatim, including spaces and dots: `GET.json`, `ACL CAT.json`,
`BF.ADD.json`, `ARSET.json`.

## Per-command JSON shape

```json
{
  "api_calls": {
    "<client_id>": [
      {
        "signature": "string — the method/function signature as a user would see it",
        "params": [
          { "name": "string", "type": "string", "description": "string (\"\" if none)" }
        ],
        "returns": { "type": "string", "description": "string (\"\" if none)" }
      }
    ]
  }
}
```

- Each client maps to an **array** of signature objects (one per overload).
- `params` order matches the signature. Exclude context/`self`/internal params
  (`ctx`, `self`, `&self`, node-redis `parser`).
- `description` is `""` when the source has no doc comment — leave it empty, don't invent.
- **Omit** any client that doesn't have the command (don't emit an empty array).
- Top-level key is always `api_calls`. No other top-level keys.

## Client IDs (must match exactly)

```
redis_py          go-redis          jedis
node_redis        ioredis           lettuce_sync
lettuce_async     lettuce_reactive  nredisstack_sync
nredisstack_async php               redis_rs_sync
redis_rs_async    redisvl
```

Note the inconsistency: **`go-redis` uses a hyphen**, everything else uses underscores.
`hiredis` (C) is excluded from the mapping. `redisvl` is the Python vector library (built on
redis-py) and rarely relevant to core/data-type commands.

## Real example (`data/command-api-mapping/GET.json`, abridged)

```json
{
  "api_calls": {
    "redis_py": [
      { "signature": "get(name: KeyT)",
        "params": [{ "name": "name", "type": "KeyT", "description": "" }],
        "returns": { "type": "ResponseT", "description": "" } }
    ],
    "jedis": [
      { "signature": "String get(final String key)",
        "params": [{ "name": "key", "type": "String", "description": "" }],
        "returns": { "type": "String", "description": "Bulk reply" } }
    ],
    "lettuce_sync": [
      { "signature": "V get(K key)",
        "params": [{ "name": "key", "type": "K", "description": "the key." }],
        "returns": { "type": "V", "description": "the value of key, or null when key does not exist." } }
    ]
  }
}
```

## Merge & validate

```bash
./build/merge-command-api-mapping.sh                       # → data/command-api-mapping.json
./build/merge-command-api-mapping.sh /tmp/out.json         # → custom path
python3 -c "import json; json.load(open('data/command-api-mapping.json'))"   # JSON sanity check
```

**Sandbox:** the merge script creates a temp file with `mktemp` under `/var/folders/...`, which
the default command sandbox blocks (`mkstemp failed … Operation not permitted`). Run the merge
with the sandbox disabled.

## Signature/returns conventions (observed from existing entries)

These vary by client; match the house style or the tables render inconsistently:

- **node_redis:** `signature` has **no return-type suffix**; `returns.type` is the **bare reply
  type, no `Promise<>`** (e.g. signature `arSet(key: RedisArgument, index: number | string, value: RedisVariadicArgument)`,
  returns `NumberReply`). See `client-quirks.md`.
- **jedis / lettuce / nredisstack:** `signature` is `ReturnType methodName(params)` (return type
  leads); `returns.type` repeats that return type.
- **redis_py:** `signature` is `method(params)` with no return; `returns.type` is the Python type.
- **go-redis:** `signature` is `MethodName(params) *XxxCmd`; `returns.type` is the `*XxxCmd` wrapper.

When in doubt, open a few existing files for that client and copy the shape exactly.
