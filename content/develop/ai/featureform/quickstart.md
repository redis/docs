The canonical quickstart now lives at [Quickstart feature workflow]({{< relref "/develop/ai/featureform/get-started/quickstart-feature-workflow" >}}), which uses the current Featureform workflow documented in this repo:

1. verify workspace access
2. confirm secret-provider setup
3. register providers
4. apply a definitions file
5. inspect graph and catalog state
6. serve from a feature view

Use [Build your first feature workflow]({{< relref "/develop/ai/featureform/tutorials/build-your-first-feature-workflow" >}}) if you want the longer guided version.
